Scoring Methodology

VulnChain scores vulnerabilities using attack path impact rather than raw CVSS severity. A CVSS 9.8 vulnerability on an isolated host scores lower than a CVSS 7.5 vulnerability on a host that sits on 47 attack paths to your domain controller.

base_score = (cvss_v3 / 10.0) x epss_score x criticality
exploit_multiplier = 1.5 if active_exploit_confirmed else 1.0
centrality_mult = 1.0 + (betweenness_centrality x 2.0)
final_score = base_score x exploit_multiplier x centrality_mult

EPSS (Exploit Prediction Scoring System) data sourced from FIRST.org. CVSS v3.1 data sourced from NIST National Vulnerability Database. Betweenness centrality computed using NetworkX directed graph traversal.
